## 8. Bài học về cách viết prompt

### 8.1. Khi viết prompt, em/nhóm cần cung cấp thông tin gì để AI trả lời tốt hơn?

```text
Cần cung cấp đầy đủ: (1) Ngữ cảnh (Context): Mục tiêu dự án và đối tượng người dùng. (2) Ràng buộc kỹ thuật (Constraints): Kiến trúc sử dụng (ví dụ: Clean Architecture) hoặc ngôn ngữ/framework (C#, .NET). (3) Đầu ra mong muốn (Output Format): Dạng danh sách, mã nguồn hoặc sơ đồ cấu trúc.
```

### 8.2. Em/nhóm đã học được gì về cách đặt câu hỏi cho AI?

```text
Việc cung cấp càng rõ ràng các yêu cầu kỹ thuật ngay từ đầu sẽ giúp AI đưa ra các đề xuất logic và bám sát kiến trúc hệ thống hơn, giảm thiểu thời gian chỉnh sửa lại cấu trúc code sau này.
```

### 8.3. Lần sau em/nhóm sẽ cải thiện prompt như thế nào?

```text
Lần tới, tôi sẽ đính kèm các lược đồ hoặc mô hình dữ liệu (ERD) ngay trong prompt để AI hỗ trợ tạo code (boilerplate code) chính xác hơn với cấu trúc 3 lớp mà tôi đang theo đuổi.
```
