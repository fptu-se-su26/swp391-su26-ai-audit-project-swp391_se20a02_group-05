#### 4.1. Prompt đã sử dụng

```text
Implement v2 pipeline tasks for the CVerify AI service. The tasks should align
with the 18-task DAG defined in CVerify.Core DagScheduler. New tasks needed:
CommitDiff (L1-003), CommitTimeline (L1-007), CommitIntent (L1-009),
Complexity (L1-010), GitBlame (L1-012), CloneDetection (L1-013),
AiGeneratedCode (L1-014), Ownership (L1-015), SkillGraph (L1-017),
TrustScore (L1-018).
```

#### 4.2. Kết quả AI gợi ý

```text
AI implemented all 10 task methods inside GitHubAnalysisOrchestrator:

- CommitDiff: Diff-First analysis mapping file paths to capability taxonomy;
  cross-validates commit messages vs structural diff patterns to detect intent conflicts.
- CommitTimeline: Temporal analysis computing commit frequency score,
  bug-to-fix ratio, refactor initiative, and working hour distribution.
- CommitIntent: LLM inference of developer intent using diff cache only;
  explicitly does not use commit messages as primary signal.
- Complexity: Deterministic L1-L6 taxonomy classification from file paths
  using a priority-ordered pattern list; computes Power Score.
- GitBlame: git blame on top-changed files producing per-file author ratios
  and overall user authorship percentage.
- CloneDetection: Heuristic detection of tutorial clones, commit bombs,
  and repos with no real development history.
- AiGeneratedCode: Detects AI-generated code signals — large single-burst
  commits, high single-touch file ratio, intent conflicts.
- Ownership: blame×0.6 + commit_ratio×0.4 minus bus_factor_penalty;
  module-level ownership breakdown.
- SkillGraph: Skill Evidence Graph from tech stack + diff capability signals + LLM.
- TrustScore: Evidence×0.40 + Ownership×0.35 + Consistency×0.25 with
  adversarial flag penalties.

Also added shared helpers: _read_meta(), _read_task_cache(), _clone_dir(),
_empty_result(), and _COMPLEXITY_PATTERNS class-level pattern table.
```

#### 4.3. Phần sinh viên/nhóm đã sử dụng từ AI

```text
- Task dispatch routing: AI added elif branches for all 10 new task types
  in the central dispatch switch.
- Full method implementations: All 10 analyze_* methods (~900 lines) were
  generated by AI based on the research spec.
- Shared helpers: _read_meta(), _read_task_cache(), _clone_dir(), _empty_result()
  used by multiple tasks to reduce duplication.
- _COMPLEXITY_PATTERNS: 30-entry ordered pattern table mapping file path
  fragments to L1-L6 capability labels and Power Score levels.
```

#### 4.4. Phần sinh viên/nhóm đã tự làm / kiểm tra

```text
  before this implementation session.
- Reviewed each task method to verify alignment with the DAG specification.
- Validated that CommitIntent does not read commit messages directly,
  only using the diff cache as primary signal.
- Confirmed the TrustScore formula weights (Evidence×0.40, Ownership×0.35,
  Consistency×0.25) match the research document §6.1.
```


#### 4.2. Kết quả AI gợi ý

```text
AI implemented three changes:

1. .env.example: Added AI_DEBUG_TOKENS=false under new section
   "# 8. TOKEN COUNT DEBUGGING" with explanatory comments including
   the API read endpoint and the security warning (dev/staging only).

2. config.py: Added ai_debug_tokens: bool = Field(False, validation_alias="AI_DEBUG_TOKENS")
   to the Settings class.

3. claude_service.py: Added a token debug writer block after the telemetry
   calculation. When ai_debug_tokens=true and job_id is present, resolves
   the temp_clones/{job_id}/ directory and appends a JSONL record containing:
   ts, task, model, prompt_tokens, completion_tokens, cache_read_tokens,
   cache_write_tokens, total_tokens, estimated_cost_usd, duration_ms,
   mismatch_flag, prompt_preview (first 300 chars when debug_mode enabled).

4. debug/invoke-pipeline.ps1: 182-line PowerShell script that accepts
   -JobId and -TaskType params, reads HMAC secret from env, signs the request,
   calls the local API, and prints a token usage summary table.
```
