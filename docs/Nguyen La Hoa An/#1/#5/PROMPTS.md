## 8. Bài học về cách viết prompt

### 8.1. Khi viết prompt, em/nhóm cần cung cấp thông tin gì để AI trả lời tốt hơn?

```text
Provide clear functional requirements, user workflows, feature descriptions, business rules, and the expected behavior of the application. Include screenshots, API documentation, or relevant code snippets when available, and specify the testing objective (e.g., functional testing with valid test cases) to receive more accurate and relevant AI-generated test cases.
```

### 8.2. Em/nhóm đã học được gì về cách đặt câu hỏi cho AI?

```text
I learned that clear, specific, and well-structured prompts produce more accurate and useful responses. Providing sufficient context, such as application requirements, testing objectives, and expected outcomes, helps AI generate more relevant test cases. I also learned that refining prompts and asking follow-up questions can improve the quality and completeness of AI-generated results.
```

### 8.3. Lần sau em/nhóm sẽ cải thiện prompt như thế nào?

```text
Next time, I will provide more detailed project requirements, user workflows, and expected outcomes in my prompts. I will include relevant screenshots, code snippets, or error messages when necessary, and refine my prompts based on AI's responses to obtain more accurate and useful suggestions. I will also continue to verify all AI-generated outputs before applying them to the project.
```
