# Changelog

## 1. Quy định ghi Changelog

File này dùng để ghi lại các thay đổi quan trọng trong quá trình thực hiện bài tập, lab, assignment hoặc project.

Nguyên tắc ghi changelog:

- Chỉ ghi những gì đã hoàn thành thật sự.
- Không ghi kế hoạch nếu chưa thực hiện.
- Mỗi thay đổi nên có ngày, nội dung, người thực hiện và minh chứng.
- Nếu có AI hỗ trợ, cần ghi rõ AI đã hỗ trợ phần nào.
- Nếu có commit GitHub, cần ghi link commit.
- Nếu có lỗi đã sửa, cần ghi rõ lỗi, nguyên nhân và cách xử lý.

## 4.2. Các chức năng chưa hoàn thành

```text
Tích hợp API thanh toán: Chưa thực hiện do ưu tiên hoàn thiện luồng dữ liệu cốt lõi (Core Logic).

```

```text
Tối ưu hóa quy trình quản lý Git, giải quyết triệt để các xung đột Merge (Merge Conflicts) trong quá trình làm việc nhóm.
```
