# Workflow Catalog

This catalog documents all AI-driven workflows operated by the `CVerify.AI` service.

## 1. Repository Evidence Analysis Workflow

Coordinates the complete code scan, classification, and evaluation of a candidate's GitHub repository.

* **Trigger**: Candidate clicks "Trigger Verification" -> C# Backend API schedules job -> Background Worker calls `/api/v1/analysis/orchestrate/stream`.
* **Execution Module**: `GitHubAnalysisOrchestrator` ([github_analysis_orchestrator.py](../../app/orchestrators/github_analysis_orchestrator.py))
* **AI Integration**: `ClaudeService.analyze_repo` ([claude_service.py](../../app/services/claude_service.py)) using system/user prompts generated by `GitHubPromptFactory`.
* **Outputs**: streams step-by-step progress events and returns a finalized structured JSON intelligence report.
* **Observability Requirements**: Correlation ID propagation across all logs, start/stop/failure event logging, token usage tracking, and cost calculation.

## 2. Conversational Repository Assistant Workflow

Enables developers and recruiters to interact with the CVerify repository intelligence agent about a scanned codebase.

* **Trigger**: User enters a message in the chat box -> React frontend POSTs to `/api/v1/chat/stream`.
* **Execution Module**: FastAPI chat stream handler ([main.py](../../app/main.py)).
* **AI Integration**: `ClaudeService.stream_chat` ([claude_service.py](../../app/services/claude_service.py)).
* **Outputs**: streams raw Markdown token text chunks.
* **Observability Requirements**: Correlation ID validation, stream failure logging, token usage tracking, and cost registry recording.
