# 03 - Agent Catalog

This document provides a detailed catalog of the agent classes defined under the `app/agents` directory. Each entry outlines the intended purpose, input/output structures, and execution status of the agent in the current production codebase.

---

## Architecture Audit Finding

> [!IMPORTANT]
> **All agents listed in this catalog are currently dead/inactive skeleton implementations.**
>
> None of these agents are instantiated, registered in any dependency injection containers, or invoked in any active runtime code path. The repository analysis is orchestrating directly via `GitHubAnalysisOrchestrator`, making a single call to Claude using prompts generated by `GitHubPromptFactory`.

---

## Agent Catalog

### 1. GitHubAgent
*   **File Path**: `app/agents/github_agent.py`
*   **Current Status**: **Dead Code / Skeleton Implementation**
*   **Intended Purpose**: Scans and analyzes a candidate's GitHub repositories to calculate overall activity and contribution metrics.
*   **Input**: `GitHubAgentInput`
    *   `candidate_id` (`UUID`)
    *   `encrypted_token` (`str`)
*   **Output**: `GitHubAgentResult`
    *   `repo_analyses` (`list`)
    *   `overall_activity_score` (`float`)
    *   `overall_contribution_score` (`float`)
*   **Dependencies**: `IAgent` interface
*   **Call Hierarchy**: Unused (no callers)

### 2. SkillExtractionAgent
*   **File Path**: `app/agents/skill_extraction_agent.py`
*   **Current Status**: **Dead Code / Skeleton Implementation**
*   **Intended Purpose**: Extract tech stacks, frameworks, and developer skill tags from raw textual profiles.
*   **Input**: `SkillExtractionInput`
    *   `text` (`str`)
*   **Output**: `ExtractedSkillsResult`
    *   `skills` (`list[str]`)
    *   `categories` (`list[str]`)
    *   `proficiency_levels` (`list[str]`)
*   **Dependencies**: `IAgent` interface
*   **Call Hierarchy**: Unused (no callers)

### 3. CvAgent
*   **File Path**: `app/agents/cv_agent.py`
*   **Current Status**: **Dead Code / Skeleton Implementation**
*   **Intended Purpose**: Extract structured skills, professional experience, and education milestones from uploaded resumes/CV text.
*   **Input**: `CvAgentInput`
    *   `submission_id` (`UUID`)
    *   `raw_text` (`str`)
*   **Output**: `CvAgentResult`
    *   `skills` (`list[str]`)
    *   `experience` (`list[str]`)
    *   `education` (`list[str]`)
    *   `completeness_score` (`float`)
    *   `raw_sections` (`dict`)
*   **Dependencies**: `IAgent` interface
*   **Call Hierarchy**: Unused (no callers)

### 4. VerificationAgent
*   **File Path**: `app/agents/verification_agent.py`
*   **Current Status**: **Dead Code / Skeleton Implementation**
*   **Intended Purpose**: Compare CV skills against GitHub analysis reports to confirm true skill proficiency.
*   **Input**: `VerificationInput`
    *   `candidate_id` (`UUID`)
    *   `repo_analyses` (`Any`)
    *   `cv_skills` (`list[str]`)
*   **Output**: `VerificationResult`
    *   `verified_skills` (`list`)
    *   `confidence_score` (`float`)
*   **Dependencies**: `IAgent` interface
*   **Call Hierarchy**: Unused (no callers)

### 5. ScoringAgent
*   **File Path**: `app/agents/scoring_agent.py`
*   **Current Status**: **Dead Code / Skeleton Implementation**
*   **Intended Purpose**: Calculate a combined, weighted confidence and competence score based on verification results and GitHub analytics.
*   **Input**: `ScoringInput`
    *   `verified_profile` (`Any`)
    *   `cv_data` (`Any`)
    *   `github_data` (`Any`)
*   **Output**: `ScoredProfile`
    *   `composite_score` (`float`)
    *   `breakdown` (`dict[str, float]`)
    *   `percentile` (`int`)
*   **Dependencies**: `IAgent` interface
*   **Call Hierarchy**: Unused (no callers)

### 6. MatchingAgent
*   **File Path**: `app/agents/matching_agent.py`
*   **Current Status**: **Dead Code / Skeleton Implementation**
*   **Intended Purpose**: Calculate candidate compatibility scores against a list of active job descriptions.
*   **Input**: `MatchingInput`
    *   `candidate_id` (`UUID`)
    *   `scored_profile` (`Any`)
    *   `jobs` (`list`)
*   **Output**: `list[MatchResult]`
    *   `job_id` (`UUID`), `candidate_id` (`UUID`)
    *   `overall_score` (`float`), `skill_match_score` (`float`), `experience_score` (`float`)
    *   `strengths` (`list[str]`), `gaps` (`list[str]`), `explanation` (`str`)
*   **Dependencies**: `IAgent` interface
*   **Call Hierarchy**: Unused (no callers)

### 7. RecommendationAgent
*   **File Path**: `app/agents/recommendation_agent.py`
*   **Current Status**: **Dead Code / Skeleton Implementation**
*   **Intended Purpose**: Generate tailored educational learning paths, CV improvements, and resume alignment summaries.
*   **Input**: `RecommendationInput`

*   **Dependencies**: `IAgent` interface
*   **Call Hierarchy**: Unused (no callers)

## AI Agent Consumption Optimization

|---|---|

| **Dependencies** | Base class: `IAgent` in [app/agents/base.py](../agents/base.py) |
| **Execution Flow** | All agent flows are uninstantiated skeletons. They directly return default empty instances of their respective response dataclasses. |
| **Common Failure Modes** | `TypeError` raised if input parameters do not match the expected dataclass type. |
| **Related Files** | [app/agents/__init__.py](../agents/__init__.py) |
| **Related Services** | None |
| **Related DTOs** | `GitHubAgentInput`, `SkillExtractionInput`, `CvAgentInput`, `VerificationInput`, `ScoringInput`, `MatchingInput`, `RecommendationInput` |
| **Related Database Tables** | None (no data access defined in agents) |
| **Related Frontend Components** | None |
